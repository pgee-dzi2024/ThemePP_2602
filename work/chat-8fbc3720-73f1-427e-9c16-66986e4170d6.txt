## Каква е причината

От конзолата се вижда, че `load_model("flower_model_opt.keras")` се проваля, защото моделът съдържа:

- **`Lambda` слой**
- в него е записана функцията **`preprocess_input`**
- при зареждане Keras не може да намери тази функция автоматично

Основното съобщение е:

> `Could not locate function 'preprocess_input'`

Това означава, че моделът е запазен с Python функция вътре в графа, а при десериализацията Keras няма откъде да я възстанови, ако тя не е подадена изрично.

## Как се оправя правилно

Най-доброто решение е:

1. **Да се дефинира/импортира `preprocess_input` преди `load_model()`**
2. **Да се подаде `custom_objects` при зареждането**
3. По възможност да се използва `compile=False`, ако само правиш предсказване

Така моделът ще се зареди коректно без да се променя самият `.keras` файл.

- `custom_objects={"preprocess_input": preprocess_input}` казва на Keras как да възстанови функцията, използвана в `Lambda` слоя
- `compile=False` избягва ненужно възстановяване на optimizer/loss/metrics, което е полезно, когато ти трябва само inference

- правиш preprocessing извън модела, както вече го правиш в `app_opt.py`

import tensorflow as tf
import numpy as np
from tensorflow.keras.models import load_model
from tensorflow.keras.preprocessing import image

# Зареждане на модела
model = load_model("flower_model.keras")

# Списък с класове (трябва да са същите като в dataset-а)
class_names = ['daisy', 'dandelion', 'rose', 'sunflower', 'tulip']

# Път до тестово изображение
img_path = "rose_2.png"  # сложи тук твоя снимка

# Зареждане и обработка
img = image.load_img(img_path, target_size=(224, 224))
img_array = image.img_to_array(img)
img_array = np.expand_dims(img_array, axis=0)

# Предсказване
predictions = model.predict(img_array)
score = tf.nn.softmax(predictions[0])

# Резултат
print("🌸 Това е:", class_names[np.argmax(score)])
print("📊 Сигурност:", round(100 * np.max(score), 2), "%")

import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

# Път до dataset-а (папката "flowers")
data_dir = "flowers"

# Зареждане на данните
train_data = tf.keras.utils.image_dataset_from_directory(
    data_dir,
    validation_split=0.2,
    subset="training",
    seed=123,
    image_size=(224, 224),
    batch_size=32
)

val_data = tf.keras.utils.image_dataset_from_directory(
    data_dir,
    validation_split=0.2,
    subset="validation",
    seed=123,
    image_size=(224, 224),
    batch_size=32
)

class_names = train_data.class_names
print("Класове:", class_names)

# Подобряване на производителността
AUTOTUNE = tf.data.AUTOTUNE
train_data = train_data.prefetch(buffer_size=AUTOTUNE)
val_data = val_data.prefetch(buffer_size=AUTOTUNE)

# Зареждане на готов модел
base_model = tf.keras.applications.MobileNetV2(
    input_shape=(224, 224, 3),
    include_top=False,
    weights='imagenet'
)

base_model.trainable = False  # НЕ обучаваме целия модел

# Добавяне на класификатор
x = base_model.output
x = layers.GlobalAveragePooling2D()(x)
x = layers.Dense(128, activation='relu')(x)
output = layers.Dense(len(class_names), activation='softmax')(x)

model = keras.Model(inputs=base_model.input, outputs=output)

# Компилиране
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

# Обучение
model.fit(train_data, validation_data=val_data, epochs=5)

# Запазване
model.save("flower_model.keras")

print("✅ Моделът е запазен като flower_model.keras")

import json
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

data_dir = "flowers"
img_size = (224, 224)
batch_size = 32
seed = 123

# Зареждане на данните
train_data = tf.keras.utils.image_dataset_from_directory(
    data_dir,
    validation_split=0.2,
    subset="training",
    seed=seed,
    image_size=img_size,
    batch_size=batch_size
)

val_data = tf.keras.utils.image_dataset_from_directory(
    data_dir,
    validation_split=0.2,
    subset="validation",
    seed=seed,
    image_size=img_size,
    batch_size=batch_size
)

class_names = train_data.class_names
print("Класове:", class_names)

# Запазваме имената на класовете
with open("class_names.json", "w", encoding="utf-8") as f:
    json.dump(class_names, f, ensure_ascii=False)

AUTOTUNE = tf.data.AUTOTUNE
train_data = train_data.cache().prefetch(buffer_size=AUTOTUNE)
val_data = val_data.cache().prefetch(buffer_size=AUTOTUNE)

# Data augmentation
data_augmentation = keras.Sequential([
    layers.RandomFlip("horizontal"),
    layers.RandomRotation(0.1),
    layers.RandomZoom(0.1),
], name="data_augmentation")

preprocess_input = tf.keras.applications.mobilenet_v2.preprocess_input

# Базов модел
base_model = tf.keras.applications.MobileNetV2(
    input_shape=(224, 224, 3),
    include_top=False,
    weights="imagenet"
)
base_model.trainable = False

# Изграждане на модела
inputs = keras.Input(shape=(224, 224, 3))
x = data_augmentation(inputs)
x = layers.Lambda(preprocess_input, name="preprocess")(x)
x = base_model(x, training=False)
x = layers.GlobalAveragePooling2D()(x)
x = layers.BatchNormalization()(x)
x = layers.Dense(128, activation="relu")(x)
x = layers.Dropout(0.3)(x)
outputs = layers.Dense(len(class_names), activation="softmax")(x)

model = keras.Model(inputs, outputs)

model.compile(
    optimizer=keras.optimizers.Adam(learning_rate=1e-3),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"]
)

callbacks = [
    keras.callbacks.ModelCheckpoint(
        "flower_model_opt.keras",
        monitor="val_accuracy",
        save_best_only=True,
        verbose=1
    ),
    keras.callbacks.EarlyStopping(
        monitor="val_accuracy",
        patience=5,
        restore_best_weights=True,
        verbose=1
    ),
    keras.callbacks.ReduceLROnPlateau(
        monitor="val_loss",
        factor=0.2,
        patience=3,
        min_lr=1e-6,
        verbose=1
    )
]

print("\n=== Етап 1: обучение само на класификатора ===")
history1 = model.fit(
    train_data,
    validation_data=val_data,
    epochs=10,
    callbacks=callbacks
)

print("\n=== Етап 2: fine-tuning само на последните слоеве ===")
base_model.trainable = True

# Замразяваме повечето слоеве
for layer in base_model.layers[:-20]:
    layer.trainable = False

model.compile(
    optimizer=keras.optimizers.Adam(learning_rate=1e-5),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"]
)

history2 = model.fit(
    train_data,
    validation_data=val_data,
    epochs=10,
    callbacks=callbacks
)

# Запазване на финалния вариант
model.save("flower_model_opt.keras")
print("✅ По-стабилният модел е запазен като flower_model_opt.keras")
print("✅ По-точен модел е запазен!")

## Какво се случва

В `flower_model_opt.keras` моделът вече съдържа:

- `Lambda(preprocess_input)` слой вътре в модела

Но в `app_opt.py` ти **също** правиш:

```python
img_array = tf.keras.applications.mobilenet_v2.preprocess_input(img_array)
```

Тоест изображението се обработва **два пъти**:

1. веднъж ръчно в `app_opt.py`
2. втори път вътре в самия модел

Това изкривява входа и много често води до резултат като:
- един клас да доминира почти винаги
- вероятности да се променят, но предсказанието да остава едно и също

---

## Защо това води точно до “dandelion” почти винаги

Моделът е обучаван да вижда изображения в една конкретна форма на данните. Ако му подадеш вече нормализирано изображение, а после то мине през `preprocess_input` още веднъж, входът става невалиден за модела.

Затова той “залепва” към един клас, който най-вероятно е научил като най-лесен отговор в тази изкривена входна среда.

---

## Правилното решение

Имаш **2 варианта**:

### Вариант 1 — най-лесният
Ако зареждаш `flower_model_opt.keras` с вътрешен `Lambda` слой, тогава:

- **махаш ръчното `preprocess_input` от inference-а**
- подаваш към модела само изображение, превърнато в array и разширено с batch dimension

### Вариант 2 — по-чистият за бъдеще
Премахваш `Lambda(preprocess_input)` от обучението и правиш preprocessing само извън модела, и тогава и обучение, и inference използват една и съща външна обработка.

За текущия ти проект **вариант 1 е най-подходящият fix**.
